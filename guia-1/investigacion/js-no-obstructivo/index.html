Javascript no intrusivo

Tenemos que el JavaScript no intrusivo es la práctica que consiste en separar completamente la funcionalidad cliente (desarrollado ocupando JS) del documento HTML; por lo que en el archivo HTML solo debería estar la línea donde se vincula el documento al script JS externo.

Si recordamos cuando empezamos en el desarrollo web se nos enseñan que existen diferentes formas de incluir JavaScript a nuestra página, siendo una de ellas colocar código JS directamente en el documento HTML que da origen a nuestra página web o haciéndolo desde un archivo externo conocido como script con la extensión “.js”; es por esta última forma, la cual es la recomendada, de donde surge la idea de JS no intrusivo pues de esta manera aseguramos dar un orden a nuestro sitio web siguiendo un modelo de capas donde las diferentes funcionalidades se integran individualmente y no todas en un solo documento para poder editar más fácilmente cada parte por separado.

Además, de esta forma aseguramos que el contenido HTML de nuestra página web se despliegue correctamente en el navegador sin detenerlo o no mostrar nada si da errores JavaScript, por si el navegador no soporta el código JS.

Podemos pensar que solamente ocupando JS no intrusivo perdemos ciertas funciones en nuestra página web, lo cual es incorrecto ya que JavaScript no permite manipular el DOM con lo cual podemos asignar funciones e incluso modificar los atributos y contenidos de elementos HTML.

Para modificar el DOM podemos utilizar JavaScript vanilla o implementar librerías que nos ayuden en esta tarea como el conocido JQuery y behaviour.
