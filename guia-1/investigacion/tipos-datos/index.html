Tipos de datos: NULL y UNDEFINED

Es importante conocer estos dos conceptos para programar eficazmente en JavaScript pues nos pueden aparecer en nuestro código y pueden ser una gran ayuda o un error indeseado si no sabemos cómo tratarlos.

Tenemos que los tipos de datos son datos primarios de JavaScript y que simbolizan el vacío o ausencia de valor, pero en realidad poseen pequeñas diferencias técnicas que no les permite ser iguales completamente.

Pues undefined es una propiedad global que representa el valor de una variable cuando no ha sido declarada o no se le ha asignado un valor.

Cuando declaramos una variable y no le damos un valor esta posee un valor undefined por lo que podemos entenderlo como el valor por defecto en una variable; también es el valor que tienen las variables que no se han declarado pero las llamamos en el código, aunque esto no se puede ver pues nos da un error de referencia.

Mientras que null es un valor que se le asigna a una variable intencionalmente para decir que está vacía por diferentes motivos según lo requiera el código, por ejemplo, necesitamos inicializar una variable pero no tenemos el valor que le vamos a dar, entonces le asignamos null para que no lo trate como undefined.

Por consiguiente, tenemos que la diferencia principal es que undefined es una propiedad que representa un valor por defecto y null un valor que se le de da a una variable intencionalmente, por lo que representan vacío pero no de la misma forma.

Nota:

Si queremos comparar estos tipos de datos, tenemos que si hacemos una comparación no estricta entre ellos, nos devuelve true. Mientras que en una comparación estricta, devuelve false.

Nota:

También cabe mencionar que se puede inicializar una variable con el tipo de datos undefined, pero no es recomendable y nos puede dar un error como al tratar de usar este tipo de dato en un JSON.

Nota:

Si ocupamos null en operaciones matemáticas, tendrá un valor de 0 ya que representa el vacío.
